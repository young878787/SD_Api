# 使用 GitHub Models API 設計服裝 - 快速開始

## 步驟 1: 獲取 GitHub Token

1. 訪問 https://github.com/settings/tokens
2. 點擊 "Generate new token" → "Generate new token (classic)"
3. Token 名稱: `outfit-designer`
4. 勾選權限: 只需要基本的 API 訪問權限（可以不勾選任何特殊權限）
5. 點擊 "Generate token" 並複製生成的 token

## 步驟 2: 配置 .env 文件

### 方法 1: 複製示例文件並編輯

```powershell
# 複製 .env.example 到 .env
Copy-Item .env.example .env

# 編輯 .env 文件，添加您的 token
# GITHUB_TOKEN=ghp_your_token_here
```

### 方法 2: 直接編輯 .env

1. 打開項目根目錄下的 `.env` 文件
2. 替換 `GITHUB_TOKEN=` 後面的內容為您的 token
3. 保存文件

**示例:**
```
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## 步驟 3: 執行腳本

### 完整功能（需要 GitHub Token）:
```powershell
python design_outfit.py
```

### 僅過濾功能（不需要 API）:
```powershell
python filter_prompt_only.py
```

## 步驟 4: 查看結果

結果會顯示在控制台，並自動保存到相應的文件

## 安全提示

⚠️ **重要**：
- `.env` 文件包含敏感信息（API Token）
- **永遠不要**提交 `.env` 文件到 Git 版本控制
- `.env` 已添加到 `.gitignore`
- 使用 `.env.example` 作為模板分享給其他開發者

## 常見問題

**Q: 忘記 Token 怎麼辦？**
A: 訪問 https://github.com/settings/tokens 查看或重新生成

**Q: Token 暴露了怎麼辦？**
A: 立即在 GitHub 設置頁面刪除該 Token，然後生成新的

**Q: 可以在命令行傳入 Token 嗎？**
A: 不推薦，這樣會在命令歷史中暴露 Token。使用 `.env` 更安全

**Q: .env 文件不存在？**
A: 複製 `.env.example` 並重命名為 `.env`，然後添加您的 Token
