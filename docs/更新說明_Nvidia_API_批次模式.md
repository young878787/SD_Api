# 🚀 更新說明 - Nvidia API 批次模式

## 📋 更新內容

### ✨ 主要改動

1. **API 切換**
   - ❌ GitHub Models API (GPT-4o)
   - ✅ Nvidia API (支持多種模型)

2. **批次模式**
   - ✅ 啟動時輸入數字決定生成次數
   - ✅ 支持 1-20 個角色批次生成
   - ✅ 自動循環生成多個角色

3. **配置更新**
   - 使用 `.env` 中的 Nvidia 配置
   - 支持自定義模型選擇

## 🔧 配置 .env

在 `.env` 文件中添加以下配置：

```env
# Nvidia API 配置
NVIDIA_API_KEY=nvapi-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NVIDIA_BASE_URL=https://integrate.api.nvidia.com/v1
NVIDIA_MODEL_NAME=meta/llama-3.1-405b-instruct
```

### 獲取 Nvidia API Key

1. 訪問 https://build.nvidia.com/
2. 註冊/登入帳號
3. 選擇模型並獲取 API Key
4. 複製 API Key 到 `.env`

### 可用模型

Nvidia API 支持多種模型，常用的有：

| 模型名稱 | 描述 |
|---------|------|
| `meta/llama-3.1-405b-instruct` | Llama 3.1 405B（推薦） |
| `meta/llama-3.1-70b-instruct` | Llama 3.1 70B |
| `meta/llama-3.1-8b-instruct` | Llama 3.1 8B（快速） |
| `mistralai/mixtral-8x7b-instruct-v0.1` | Mixtral 8x7B |
| `google/gemma-2-27b-it` | Gemma 2 27B |

## 🚀 使用方式

### 1. 基本使用

```bash
python auto_character_generator.py

# 輸入提示：
請輸入要生成的角色數量（1-20）: 3

# 程序將自動生成 3 個不同的角色
```

### 2. 執行流程

```
1. 輸入數量（例如：5）
   ↓
2. 檢查 SD WebUI 連接
   ↓
3. 循環 5 次：
   - 隨機抽獎角色類型
   - Nvidia API 設計角色
   - 組合完整 Prompt
   - 發送到 SD 生成
   - 自動保存圖片
   ↓
4. 顯示所有生成的圖片
```

### 3. 批次生成示例

```bash

🎯 批次 2/3 - 成熟女性
🎯 批次 3/3 - 可愛少女
🎉 全部完成！共生成 3 張圖片
```


## 📊 批次模式特點

### ✅ 優勢

1. **一次運行，多次生成**
   - 無需重複啟動程序
   - 自動化處理整個批次

2. **錯誤處理**
   - 單個批次失敗不影響其他
   - 自動跳過失敗的批次

3. **進度顯示**
   - 顯示當前批次進度（1/5, 2/5...）
   - 實時反饋生成狀態

4. **統一總結**
   - 批次結束後顯示所有生成的圖片
   - 一目了然查看成果

### 📈 性能考慮

| 數量 | 預估時間 | 說明 |
|------|---------|------|
| 1-3 | 2-6 分鐘 | 適合快速測試 |
| 4-5 | 8-10 分鐘 | 推薦日常使用 |
| 6-10 | 12-20 分鐘 | 批量生成 |
| 11-20 | 22-40 分鐘 | 大量生成 |

**注意：** 時間取決於：
- SD 生成速度
- Nvidia API 響應時間
- 網絡連接速度
- 圖片尺寸和採樣步數

## ❓ 常見問題

### Q: Nvidia API 比 GitHub Models 快嗎？

A: 速度取決於：
- 選擇的模型（8B < 70B < 405B）
- 網絡延遲
- API 配額和限制

### Q: 批次生成可以中斷嗎？

A: 可以！按 `Ctrl+C` 中斷，已生成的圖片會保留。

### Q: 如何選擇模型？

A: 
- **快速生成**：llama-3.1-8b-instruct
- **平衡選擇**：llama-3.1-70b-instruct
- **最高質量**：llama-3.1-405b-instruct（推薦）

### Q: 批次太多會有問題嗎？

A: 建議一次不超過 10 個，避免：
- API 配額超限
- SD 生成隊列過長
- 等待時間過久
