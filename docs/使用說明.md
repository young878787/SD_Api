## 環境配置

### 系統要求

- Python 3.7+
- Windows / Mac / Linux

### 安裝依賴

用戶需要自己安裝以下 Python 套件：

```powershell
# 安裝所需套件
pip install openai python-dotenv

# 或者一次性安裝所有
pip install openai python-dotenv
```

### 配置 GitHub Token

#### 方法 1：通過 .env 文件（推薦）

1. 打開 `.env` 文件
2. 添加您的 GitHub Token：
   ```
   GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
3. 保存文件

#### 方法 2：通過 PowerShell 環境變數

```powershell
# 臨時設置（只在當前會話有效）
$env:GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# 之後執行腳本
python design_outfit.py
```

### 獲取 GitHub Token

1. 訪問 https://github.com/settings/tokens
2. 點擊 "Generate new token" → "Generate new token (classic)"
3. 配置 Token：
   - 名稱：`outfit-designer`
   - 過期時間：自定義
   - 權限：無需特殊權限（可為空）
4. 點擊 "Generate token" 並複製

## 常見問題

### Q: 為什麼使用 .env 而不是環境變數？

**A:** .env 文件的優點：
- 便於管理（集中管理所有配置）
- 不會暴露在命令歷史中
- 便於在團隊中分享 `.env.example`
- 符合行業標準做法

### Q: .env 文件應該上傳到 GitHub 嗎？

**A:** 不應該。.env 包含敏感信息：
- 使用 `.env.example` 作為模板分享
- 將 `.env` 添加到 `.gitignore`
- 每個開發者有自己的 `.env` 文件

### Q: 如果 Token 過期怎麼辦？

**A:** 生成新的 Token：
1. 訪問 https://github.com/settings/tokens
2. 刪除舊的 Token
3. 生成新的 Token
4. 更新 `.env` 文件

### Q: 可以在服務器上運行嗎？

**A:** 可以，但要注意：
- 確保 `.env` 文件安全
- 使用環境變數而不是硬編碼 Token
- 定期輪換 Token

### Q: 支持批量處理多個 JSON 文件嗎？

**A:** 目前不支持，但可以：
1. 複製腳本進行修改
2. 添加循環遍歷多個文件
3. 修改輸出文件名避免覆蓋
